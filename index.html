New project & update on Kaleido!

Published on November 3, 2025 by Lynn Hogen-Esch

Hey folks! I've been busy working on a few new things. First, I'm starting work on a tower defense/arcade shooter game (no title yet, I'll be calling it Druid Defense for now) in which you defend a big tree against hordes of robots! As I typically am, I am the UI/UX lead for this project, and I'm using this project to figure out some better ways of having dynamic UI, to keep the player in the game better. Namely I'm working on making tower placement a radial menu rather than a hotbar, so as to have the least amount of HUD that stays constantly visible as possible.

Additionally, I've been reworking the overworld UI for Kaleido! I've figured out the layouts for the inventory screen and the new party screen. I've also reworked the structure of the overworld UI, see the table below for the comparison. You might notice if you've been following the development of the UI that we no longer have a status screen. This is because in the new layout, character stats are viewable on the party screen! The switch button lets you swap characters in and out of your active party, and if you aren't switching a character, it'll display the stats of the character you have selected. You'll also notice that the included screenshots of the overworld UI do not include the button for the map. This is because the map was added to the plan for the overworld UI after these screenshots were taken. I've actually recently decided to pivot to a different part of the UI, though I won't be able to speak much on what I'm pivoting to as it would be spoiling the opening of the game. So, if I speak on my contribution to Kaleido, it will likely either be abstracted enough that it wouldn't spoil anything, or about some other contribution. See y'all next week!

Resuming work on Kaleido!!

Published on October 20, 2025 by Lynn Hogen-Esch

I've recently begun working on Kaleido again! Kaleido is a JRPG with fighting game elements that I first began work on during the summer of 2023. Once MSU classes started up in the fall of that year, the team kinda stopped working on it. This was, in hindsight, for the best, as it allowed us to develop our skills further before resuming work on this project. I am the UI/UX team lead for Kaleido, so I've been working on the UI layouts. When we came back to pre-production of Kaleido, I decided to do a full reset on the UI as what we had was made when I was significantly less competent in my UI/UX design skills. Kaleido is a very large scope game, the team is currently working toward making just the prologue (we're aiming for 3-4 hours of gameplay) as a demo to show to publishers. Currently we are all just working on Kaleido as a side gig while we are all in college, but we eventually plan on incorporating as a full studio once we have the funding to support that. We're also planning on starting a devlog for the project, which will likely be on youtube. While Kaleido will have its own devlogs, I'll be including my work on Kaleido in my weekly devlogs! See y'all next week!
